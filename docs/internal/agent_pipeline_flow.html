📥 Debbie's Ad Hoc Intel Path

Spot a competitor on Reddit, LinkedIn, or social media? Drop it in docs/internal/competitive/INTAKE.md with name, URL, and notes. The Competitive Intel agent processes all pending entries at the start of every scan and routes findings to the right agents via pipeline tracker tags.

What Flows Where

🔍 Comp Intel Produces

Scans the competitive landscape and processes Debbie's INTAKE.md tips. Creates pipeline items tagged RON: (feature gaps), MADISON: (messaging angles), GINA-REVIEW: (architecture concerns), BROCK-REVIEW: (security comparisons). Report in docs/internal/competitive/.

🔨 Ron Consumes

Reads RON: tasks from pipeline tracker. Builds features to close competitive gaps. Also reads Meg (QA), Brock (security), and Gina (architecture) findings. Outputs code + COMPLETED.md updates.

🧪 Meg + 🛡 Brock Validate

Meg tests Ron's code for correctness. Brock scans for vulnerabilities and picks up BROCK-REVIEW: items from competitive intel. Both feed findings back to Ron.

🏛 Gina Evaluates

Reviews GINA-REVIEW: items -- evaluates competitor approaches for architectural merit. Reviews Ron's product code for quality. Feeds back to Ron (fixes) and Brock (security deep-dives).

📝 John Documents

Writes user-facing docs for features Meg has verified as working. CLI references, MCP tool catalogs, quickstart guides, changelogs. Outputs to ron_skills/*/docs/.

📣 Madison Markets

Picks up MADISON: messaging angles from competitive intel. Writes blog drafts and promo copy positioning Lore's differentiators. Never names competitors directly.

📊 Jacqueline Synthesizes

Reads ALL agent outputs including competitive scan reports. Produces the daily executive dashboard and weekly product roadmap. Surfaces competitive findings, handoff status, and action items for Debbie.

🔭 Scout Researches

Reads competitive intel to avoid proposing overlapping products. Uses competitor gaps to find adjacent opportunities. Feeds new OPPs into pipeline for Gina's architecture review and Debbie's triage.
