## Summary

This QA run covered 5 commits since the last Meg report (8780aa9..HEAD). The changes include:

1. **Plugin .mcp.json PRO defaults (MEG-037)**: Both loreconvo-plugin and loredocs-plugin .mcp.json files now set PRO=1. This was intended for internal agent use but CLAUDE.md TODOs #2 and #3 explicitly require these to default to empty string for public users. The dev .mcp.json (loreconvo/.mcp.json) having PRO=1 is correct -- it is internal only. The -plugin variants ship to external users and must be free tier.

2. **Plugin.json hook structure fix**: The LoreConvo plugin.json hooks were flattened from a nested `{hooks: [{type, command}]}` structure to a flat `[{type, command}]` array. This is the correct Claude plugin hook format. SessionStart now has 2 hooks (setup + on_session_start), SessionEnd has 1 hook with timeout=5.

3. **Fallback scripts**: Two new scripts added for agent use when MCP tools are unavailable:
   - `scripts/save_to_loreconvo.py` -- save, read, and search LoreConvo sessions
   - `scripts/query_loredocs.py` -- list, info, search, and add-doc for LoreDocs vaults
   - Both also copied to their respective `ron_skills/*/scripts/` directories. Clean implementations with parameterized SQL, UUID generation, auto DB discovery.

4. **README onboarding improvements**: Both LoreConvo and LoreDocs READMEs now include: verify installation section, recommended CLAUDE.md setup snippet, Cowork user guidance, and fallback script documentation.

5. **COWORK_RESTORE.md rewrite**: Complete rewrite with CLAUDE.md-first approach, corrected tool names (was vault_search/vault_list_sessions, now get_recent_sessions/search_sessions), and practical examples.

6. **License update**: loreconvo-plugin plugin.json updated from MIT to BSL-1.1.

## Findings

### MEG-037: HIGH - Public plugin .mcp.json files default to PRO tier

**Location**: `ron_skills/loreconvo-plugin/.mcp.json`, `ron_skills/loredocs-plugin/.mcp.json`

**Description**: Both public-facing plugin .mcp.json files have `LORECONVO_PRO: "1"` / `LOREDOCS_PRO: "1"`. These are the files that external users would install. CLAUDE.md TODOs #2 and #3 explicitly require setting these to `""` (empty string) so public users get the free tier by default.

**Impact**: If these .mcp.json files ship as-is, all external users would automatically get Pro tier features without paying. This undermines the entire freemium business model.

**Fix**: Set `"LORECONVO_PRO": ""` in loreconvo-plugin/.mcp.json and `"LOREDOCS_PRO": ""` in loredocs-plugin/.mcp.json. The dev .mcp.json (loreconvo/.mcp.json) can stay at "1" since it is internal only.

**Severity**: HIGH -- directly impacts revenue strategy. Must be fixed before any marketplace distribution.

**Test coverage**: test_mcp_json_pro_defaults.py documents the current state. Tests will need updating when the fix lands.

### MEG-036: MEDIUM - Cross-file test pollution (unchanged from 2026-04-02)

**Location**: ron_skills/sql_query_optimizer/api/tests/test_credits_concurrency.py

**Description**: Two tests fail due to module-level DATA_DIR caching. Correctly isolated replacements exist in test_credits_isolation.py.

**Severity**: MEDIUM -- does not affect production code, but undermines test suite reliability.
